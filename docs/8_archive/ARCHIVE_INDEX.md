## Retention Policy

**Current Policy**: Indefinite retention in `8_archive/` directory

**Rationale**:
- Git history preserves all content
- Archive provides easier access than git blame/log
- Disk space is not a constraint
- Historical context valuable for understanding evolution

**Future Consideration**: 
- After project completion (post-1.0.0), conduct review to identify files for permanent removal
- Candidates for removal: test artifacts, progress trackers, duplicate summaries

## Usage Guidelines

### ‚úÖ Good Uses for Archive Files
- Understanding historical design decisions
- Tracking evolution of specific features
- Audit purposes (understanding "why did we do this?")
- Comparative analysis (before/after refactoring)

### ‚ùå Bad Uses for Archive Files
- Reference for current development (use current docs instead)
- Copy-pasting code examples (may be outdated)
- Following outdated workflows (use current guides)
- Citing in new documentation (link to current docs)
